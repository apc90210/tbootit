# Receipt and Price Tag Templates Requirements

## 1. Товарно-гарантийный чек
Чеки оформляются после добавления товаров в корзину продаж (Stage 04E-R4).
Они печатаются на A4 / Letter, а не на термопринтере.

**Содержание чека:**
1. Реквизиты организации:
   - Название
   - ИНН
   - Адрес
   - Телефон
2. Заголовок "Товарный чек № <номер> от <дата>"
3. Таблица с товарами:
   - №
   - Наименование
   - Код товара (в будущем SKU / Barcode)
   - Ед. изм.
   - Количество
   - Цена
   - Сумма
4. Подвал с итогами:
   - Итого
   - Предоплата (по умолчанию 0.00 для MVP)
   - К оплате
   - Количество наименований
5. Подписи:
   - Продавец (с указанием ФИО по умолчанию из настроек организации)
   - Покупатель (с меткой "Частное лицо" по умолчанию)
6. Блок гарантии:
   - Если гарантия есть: "На все Б/У товары предоставляется гарантия N дней." + место для подписи покупателя.
   - Если гарантии нет: "Товар продаётся без гарантии, в том состоянии, в котором есть. Покупатель внимательно осмотрел товар при покупке." + место для подписи.

*Примечание:* Данные реквизитов организации загружаются из Core API (`organization_settings`), которые можно настроить через UI в разделе "Настройки".

## 2. Ценник (Термопринтер 58×40 мм)
Печать ценников доступна напрямую из карточки товара или таблицы товаров, если товар имеет статус "В наличии".

**Требования к верстке:**
1. Размер страницы (CSS `@page`): `size: 58mm 40mm; margin: 0;`
2. Отступы внутри ценника не должны превышать 2mm для максимизации полезного пространства.

**Содержание ценника:**
1. Название магазина (Например, "Магазин Техноребут").
2. Категория или тип товара (уменьшенный шрифт).
3. Наименование товара (крупно, обрезается, если не влезает в 2-3 строки).
4. Место под штрихкод (placeholder в MVP).
5. Код товара (внутренний ID или SKU).
6. Цена продажи (выделено, самый крупный шрифт на ценнике).

## 3. Barcode Future Foundation (Фундамент для штрихкодов)
В текущем MVP (Stage 04E) сканирование и печать штрихкодов не реализованы, но архитектура спроектирована с учетом их внедрения на будущих этапах.

**Планируемый функционал:**
- Поле `product.barcode` в БД Core.
- В ценник 58×40 мм вместо placeholder будет рендериться штрихкод (EAN-13 или Code-128).
- В таблице товаров и корзине появится поле ввода "Сканировать штрихкод" для быстрого поиска и добавления в чек.
- В чеке в столбце "Код" будет выводиться значение `barcode` вместо внутреннего ID, если оно задано.
