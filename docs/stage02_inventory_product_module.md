# Stage 02: Товарный модуль и остатки (Inventory & Product Module Hardening)

## Цели этапа
Усиление товарного модуля для поддержки реального учета остатков, истории изменений и подготовки к публикации на внешних площадках (Сайт, Авито).

## Выполненные задачи
- Расширена модель `Product`: добавлены поля для остатков (`quantity`, `reserved_quantity`), цен (`min_price`, `market_price`), примечаний и признаков публикации.
- Созданы модели `ProductEvent` и `StockMovement` для аудита любых изменений товара и истории движений остатков.
- Реализованы новые API-ручки (`/api/products/{id}/details`, `/api/products/{id}/stock-adjustment`, `/api/products/{id}/site-publication` и др.).
- Добавлена фильтрация товаров (`q`, `status`, `brand`, `storage_location`).
- Обновлен Admin Shell (`index.html`): добавлен фильтр товаров, модальное окно деталей товара с таблицами истории и формами корректировки остатков/публикации. Все тексты и элементы интерфейса полностью русифицированы.
- Обновлена документация (`manual_test.md`, `database.md`, `api_contract.md`).
- Внедрен механизм ad-hoc авто-миграции для SQLite при запуске (в `main.py`), чтобы добавить новые поля в таблицу `products` без потери старых данных.

## Тестирование
- Написаны тесты (`test_products.py`) для проверки получения фильтров/мета-данных, корректировки остатков и установки флагов публикаций.
- Все Docker-контейнеры пересобраны и успешно запущены.
- Успешно выполняется `Invoke-RestMethod` к обновленным эндпоинтам.

Модуль товаров теперь является надежным ядром для отслеживания инвентаря.
