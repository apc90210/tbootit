# Сценарии ручного тестирования

1. Добавьте товар (через Admin Shell API или Docs), укажите количество и цены.
2. Загрузите фото через Docs `POST /api/products/{id}/photos`.
3. Откройте модальное окно товара, нажав на его название в Admin Shell.
4. Выполните корректировку остатков через форму. Убедитесь, что остаток изменился, а запись появилась в Истории движений.
5. Установите галочки публикации для сайта и Авито. Проверьте, что появились зеленые бейджи "Сайт" и "Авито".
6. Используйте фильтр товаров (поиск, бренд, локация, статус) для поиска конкретных товаров.
7. Измените статус товара на `Продан` через Admin Shell.
8. Создайте клиента.
9. Создайте заявку на ремонт.
10. Проведите продажу.
11. Проверьте структуру БД в Admin Shell (или по адресу `http://127.0.0.1:8000/api/admin/db/schema`).
12. Проверьте журнал действий в Admin Shell (или по адресу `http://127.0.0.1:8000/api/admin/audit-log`).
13. Перезапустите Docker: `docker compose down`, затем `docker compose up`. Убедитесь, что данные сохранились.

Внимание: Admin Shell доступен по адресу `http://127.0.0.1:8011`.
