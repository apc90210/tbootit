# Stage 04E — Sales UI MVP

В рамках `Stage04E` реализована продажа товаров через пользовательский интерфейс модуля `inventory-sales-module`.

## Архитектура
Продажа идет **строго через Core API** (`POST http://core:8000/api/sales/`).
Прямой доступ к базе данных (`sqlite3`, `SQLAlchemy`) из `inventory-sales-module` запрещен и защищен тестами.

## Пользовательский процесс (UI Flow)
1. Оператор видит список товаров в статусах "В наличии" (`in_stock`) или "В резерве" (`reserved`).
2. Напротив таких товаров активна кнопка **"Продать"** (красная).
3. При нажатии открывается форма новой продажи (`GET /sales/new?product_id=X`), где отображается цена каталога.
4. Оператор может изменить итоговую цену продажи, выбрать способ оплаты (Наличные, Карта, Перевод и т.д.) и добавить примечание.
5. При нажатии **"Подтвердить продажу"** отправляется запрос в Core API.
6. Core API создает продажу, меняет статус товара на `sold` ("Продан") и сохраняет событие в историю товара (`sale_completed`).
7. Пользователь перенаправляется на страницу успешной продажи (`GET /sales/{sale_id}`), где указан номер транзакции и детали.
8. Повторная продажа проданного товара недоступна (кнопка деактивирована).

## Ограничения MVP (Out of Scope for Stage 04E)
- Отмена продажи через UI (будет добавлена позже).
- Возврат товара через UI.
- Массовые продажи (bulk sales).
- Продажа нескольких товаров в одном чеке (корзина).
- Печать ценников и чеков (Stage 04F).
