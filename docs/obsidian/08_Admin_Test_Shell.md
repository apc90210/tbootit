# 08 Admin/Test Shell

## Назначение

Admin/Test Shell — простая внешняя оболочка для тестирования Core API.

Это не финальная админка. Это инструмент для проверки и первичной работы с ядром.

## Главные правила

- работает только через HTTP API;
- не подключается к БД;
- не импортирует модели Core;
- может быть заменена будущими внешними модулями.

## Разделы Shell

### Dashboard

Показывает:

- Core health;
- количество товаров;
- количество клиентов;
- количество ремонтов;
- количество продаж.

### Products

Функции:

- список товаров;
- добавление;
- просмотр карточки;
- изменение статуса;
- загрузка фото;
- просмотр фото.

### Customers

Функции:

- список клиентов;
- добавление клиента.

### Repairs

Функции:

- список ремонтов;
- создание ремонта;
- изменение статуса.

### Sales

Функции:

- список продаж;
- проведение продажи.

### DB Structure

Функции:

- список таблиц;
- поля таблиц;
- количество записей.

### Admin

Функции:

- seed data;
- backup;
- audit log;
- dev reset.

## UI-подход

На MVP допустим простой интерфейс:

- одна страница;
- HTML forms;
- таблицы;
- минимум CSS;
- понятные кнопки.

Цель — проверить бизнес-логику, а не дизайн.
