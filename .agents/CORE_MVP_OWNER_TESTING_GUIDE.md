# OWNER TESTING GUIDE — Core MVP

## Цель

Пользователь вручную проверяет, что Core MVP реально удобен как первый прототип.

## Запуск

В PowerShell:

```powershell
Set-Location C:\tbootit
docker compose up --build
```

Открыть:

```text
http://127.0.0.1:8010
```

Также проверить API:

```text
http://127.0.0.1:8000/docs
```

## Проверка 1 — Dashboard

Проверить:

- видит ли оболочка Core API;
- отображаются ли счетчики;
- нет ли ошибок подключения.

## Проверка 2 — Добавление товара

Создать тестовый товар:

```text
Название: Ноутбук Lenovo ThinkPad T480
Категория: Ноутбуки
Бренд: Lenovo
Модель: T480
Серийный номер: TEST-T480-001
Состояние: БУ, рабочий
Цена закупки: 15000
Цена продажи: 25000
Статус: in_stock
Место хранения: Склад 1
```

Проверить:

- товар появился в списке;
- открывается карточка;
- поля отображаются правильно.

## Проверка 3 — Фото

Добавить фото к товару.

Проверить:

- фото загрузилось;
- фото видно в карточке;
- ссылка на фото открывается.

## Проверка 4 — Статус товара

Изменить статус товара:

```text
in_stock → reserved
reserved → in_stock
```

Проверить:

- статус меняется;
- в audit log появляется запись.

## Проверка 5 — Клиент

Создать клиента:

```text
Имя: Иван Тестовый
Телефон: +7 900 000-00-00
Комментарий: Тестовый клиент
```

Проверить:

- клиент появился в списке.

## Проверка 6 — Ремонт

Создать ремонт:

```text
Клиент: Иван Тестовый
Устройство: HP LaserJet 2055dn
Серийный номер: TEST-HP-001
Проблема: Не захватывает бумагу
Статус: new
```

Поменять статус:

```text
new → diagnostics → in_progress → ready
```

Проверить:

- ремонт отображается;
- статусы меняются;
- данные сохраняются.

## Проверка 7 — Продажа

Создать или выбрать товар со статусом `in_stock`.

Провести продажу.

Проверить:

- продажа появилась в списке;
- товар стал `sold`;
- audit log содержит событие продажи.

## Проверка 8 — Структура БД

Открыть раздел DB Structure.

Проверить:

- видны таблицы;
- видны поля;
- видно количество записей.

## Проверка 9 — Администрирование

Проверить:

- seed data;
- backup;
- audit log.

Dev reset использовать только если нужно очистить тестовые данные.

## Проверка 10 — Сохранение данных

Остановить:

```powershell
docker compose down
```

Запустить:

```powershell
docker compose up
```

Проверить, что товары, клиенты, ремонты и продажи остались.

## Итоговая приемка

Если все работает, зафиксировать:

```text
OWNER_MANUAL_TEST_PASSED
```

Если есть замечания, записать:

```text
1. Что не работает.
2. Где нажимал.
3. Что ожидал.
4. Что произошло.
5. Скриншот/текст ошибки, если есть.
```
