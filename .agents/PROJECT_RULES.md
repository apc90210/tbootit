# tbootit Project Rules

## ель

роект tbootit ведётся поэтапно. аждый этап должен быть понятен, проверяем и фиксироваться в git.

## азовые правила

1. е менять сразу много несвязанных частей проекта.
2. аждый этап выполнять в отдельной ветке, кроме первичной подготовки.
3. еред изменениями проверять:
   - текущую ветку;
   - git status;
   - наличие незакоммиченных файлов.
4. осле изменений обязательно:
   - показать список изменённых файлов;
   - выполнить доступные проверки;
   - сделать краткий отчёт.
5. Секреты, пароли, токены и приватные ключи не коммитить.
6. оги и временные файлы не коммитить.
7. се промты, отчёты и правила хранить в .agents/.
8. абочая ветка main должна оставаться стабильной.

## ормат отчёта этапа

- то сделано
- акие файлы изменены
- акие проверки выполнены
- то осталось
- ожно ли принимать этап
