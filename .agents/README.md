# .agents — Техноребут Core MVP

Набор правил для работы агента в проекте `C:\tbootit`.

## Главная цель

Создать полностью рабочий прототип ядра системы «Техноребут»:

- Core API как центральный владелец данных.
- База данных внутри Core-модуля.
- Фото и файлы внутри Core-модуля.
- Простая внешняя оболочка для тестирования и администрирования.
- Остальные будущие модули работают только через HTTP API.

## Основное архитектурное правило

`Core API + DB + Storage = единое ядро системы.`

Внешние модули не имеют прямого доступа к БД и файловому хранилищу. Они используют только публичные/служебные HTTP endpoints Core API.

## Рабочий каталог

```powershell
C:\tbootit
```

Если проект пустой, агент создает структуру с нуля. Если файлы уже существуют, агент сначала анализирует состояние и вносит изменения минимально.

## Стиль разработки

1. Сначала большой рабочий модуль.
2. Потом внутренние самопроверки.
3. Потом отчет.
4. Если требуется — ручное owner-тестирование пользователем.
5. После принятия — следующий модуль.

## Запрещено

- Не подключать внешние модули напрямую к БД.
- Не выносить фото в отдельный модуль на MVP.
- Не усложнять безопасность раньше времени.
- Не начинать сайт, Авито или продажи как отдельные полноценные продукты до готовности Core MVP.
- Не скрывать ошибки тестов.
- Не оставлять проект в состоянии, когда `docker compose up --build` не запускается.

## Разрешено для MVP

- Простая авторизация через dev-token.
- SQLite как встроенная БД прототипа.
- Хранение файлов на volume.
- Простая HTML/JS внешняя оболочка без сложного frontend-фреймворка.
- Простые seed-данные.
- Админские функции разработки: просмотр схемы БД, экспорт, backup, reset dev data.

## Главный результат первого этапа

```text
docker compose up --build
```
