# PROJECT_CONSTITUTION — Техноребут

## 1. Назначение системы

«Техноребут» — ИТ-система магазина и сервисного центра по ремонту, продаже и учету компьютерной и оргтехники, преимущественно БУ-техники.

Система должна поддерживать:

- учет товаров;
- учет остатков;
- фото товаров;
- ремонтные заявки;
- клиентов;
- продажи;
- подготовку товаров к публикации на сайте;
- подготовку товаров к публикации на Авито;
- дальнейшее подключение внешних модулей.

## 2. Архитектурная основа

Core API является центром системы.

Внутри Core:

- API;
- БД;
- файловое хранилище;
- бизнес-логика;
- журнал действий;
- административные endpoints;
- API-контракты для внешних модулей.

## 3. Модульность

Первый этап:

```text
Core MVP + Admin/Test Shell
```

Следующие этапы:

```text
Public Site
Repairs UI
Sales UI
Avito Module
Telegram/notification module
Analytics/reporting module
```

## 4. Контракт между модулями

Единственный официальный способ взаимодействия внешних модулей с данными — HTTP API Core.

Внешний модуль может запускаться в том же Docker Compose, отдельно на локальной машине, на другом сервере, как бот, как сайт или как отдельная админка.

## 5. MVP-подход

Для первого прототипа приоритет:

1. Работает полностью.
2. Можно тестировать руками.
3. Данные сохраняются.
4. Фото загружаются.
5. Структура БД просматривается.
6. Ошибки видны.
7. Код понятен и расширяем.

Красота интерфейса, сложная безопасность, роли, платежи, касса и интеграции откладываются.
