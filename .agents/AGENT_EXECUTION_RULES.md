# AGENT_EXECUTION_RULES

## Роль агента

Агент работает как senior backend/fullstack developer и технический архитектор проекта «Техноребут».

## Рабочий цикл каждого модуля

1. Preflight-анализ.
2. Реализация.
3. Внутренние самопроверки.
4. Исправление найденных ошибок.
5. Итоговый отчет.
6. Инструкции для ручного тестирования, если требуется.

## Preflight

Перед изменениями агент должен проверить:

```powershell
Set-Location C:\tbootit
git status
git branch
Get-ChildItem
```

Если репозиторий пустой — создать проект с нуля. Если есть файлы — не удалять их без причины.

## Обязательные проверки после реализации

Минимум:

```powershell
docker compose config
docker compose up --build
```

Затем проверить:

```text
GET /health
GET /api/version
GET /api/admin/db/schema
GET /api/products
Admin Shell открывается
```

Если есть тесты:

```powershell
pytest
```

## Git-правила

После успешной реализации:

```powershell
git status
git add .
git commit -m "Implement Core MVP prototype"
```

Без явной команды пользователя не выполнять опасные операции с remote.

## Отчет агента

В конце агент должен создать отчет:

```text
reports/core_mvp_implementation_report.md
```

В отчете указать:

- что создано;
- структура проекта;
- какие endpoints работают;
- как запустить;
- как протестировать;
- какие проверки выполнены;
- какие ограничения MVP остались;
- что делать следующим модулем.

## Правило ошибок

Если проверка не прошла:

1. Указать ошибку.
2. Попробовать исправить.
3. Повторить проверку.
4. Если не удалось — оставить точный отчет и команды воспроизведения.

## Нельзя

- Нельзя оставлять нерабочий Docker без отчета.
- Нельзя обещать, что проверено, если команда не запускалась.
- Нельзя переписывать архитектуру без причины.
- Нельзя подключать внешние модули к БД напрямую.
- Нельзя хранить фото во внешнем модуле.
