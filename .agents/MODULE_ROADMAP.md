# MODULE_ROADMAP — модульный план разработки Техноребут

## Общая логика

Система строится крупными модулями. Каждый модуль должен давать рабочий результат, который можно запустить, проверить и принять.

## Stage 00 — Architecture & Rules

Цель: зафиксировать архитектуру, правила агентов и документацию.

Результат:

- `.agents` правила;
- Obsidian-документация;
- roadmap;
- MVP scope;
- Core API contract baseline.

---

## Stage 01 — Core MVP Big Module

Цель: создать полностью рабочее ядро и простую внешнюю оболочку для тестирования.

Состав:

- Docker Compose;
- Core API;
- встроенная БД;
- файловое хранилище;
- базовая схема данных;
- CRUD товаров;
- CRUD категорий;
- CRUD клиентов;
- ремонтные заявки;
- продажи;
- фото товаров;
- журнал действий;
- endpoints просмотра структуры БД;
- dev-admin endpoints;
- внешняя оболочка Admin/Test Shell.

Критерий готовности:

```text
docker compose up --build
Core API работает
Admin Shell работает
можно добавить товар
можно посмотреть товар
можно загрузить фото
можно посмотреть структуру БД
можно создать клиента
можно создать ремонт
можно провести продажу
данные сохраняются после перезапуска
```

## Stage 01A — Internal Self-Check

Цель: проверить Core MVP внутренними тестами агента.

Проверки:

- запуск Docker;
- health endpoints;
- API smoke;
- CRUD smoke;
- file upload smoke;
- DB schema introspection smoke;
- Admin Shell smoke;
- persistence after restart;
- audit log smoke.

Результат:

```text
reports/core_mvp_self_check_report.md
```

---

## Stage 01B — Owner Manual Testing

Цель: пользователь вручную проверяет MVP.

Пользователь проверяет:

- открывается ли оболочка;
- удобно ли добавить товар;
- видны ли остатки;
- загружается ли фото;
- видна ли структура БД;
- можно ли создать ремонт;
- можно ли провести продажу;
- понятно ли, что делать дальше.

Результат:

```text
OWNER_MANUAL_TEST_PASSED
```

или список замечаний.
